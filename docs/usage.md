# 自用食谱网站使用说明

## 打开方式

直接双击 `index.html`，或用浏览器打开 `E:\recipe-site\index.html`。

## 首页与导航

- 桌面端顶部导航提供“菜谱库、实用工具、AI 成谱”三个主要方向；首页首屏以“AI 生成菜谱”为主操作，手动新增作为次级操作。
- “今天吃什么、配一桌菜、按食材找菜”直接显示在首屏；厨具筛选和购物清单收纳在“更多实用工具”中。
- 外观、备份、昵称和日志统一收纳到“我的设置与数据”，减少首页按钮干扰，但原有功能保持不变。
- 手机端底部固定导航包含“首页、菜谱、AI 成谱、工具、我的”。“菜谱”会定位到菜谱库，“工具”会定位到今晚决策区，“我的”会展开设置与数据入口。
- 网站遵循系统的“减少动态效果”设置；关闭或减弱动画不会影响导航和功能操作。
- 桌面端支持按钮扫光、卡片局部光影、图片轻微放大、菜单展开和点击反馈；这些效果仅用于状态提示，不改变点击结果。触屏设备会自动关闭鼠标跟随光影以保证流畅度。

## 记录内容

每个做法版本可以记录：

- 菜名
- 所属世界（现实中的饭 / 二次元中的饭 / 黑暗料理 / 存在于幻想中的饭，可自行增删改）
- 厨师/来源
- 做法版本/特点
- 配图链接
- 菜品介绍
- 大概成本
- 适合什么时候吃
- 大概耗时
- 教学视频链接
- 具体做法
- 所需工具（选填，空格分隔，如：炒锅 烤箱）

## 编辑方式

- 点击“新增菜谱”后默认进入“小白引导模式”，按“基本信息、食材、做法、确认”四步填写；只有菜名，以及“做法步骤或视频链接”是保存所必需的。
- 不确定的介绍、成本、耗时和配图可以先跳过；确认页会集中提示还缺什么。
- 食材既可以逐条添加，也可以点击“粘贴一整段”自动拆分；做法每行写一步，页面会显示识别出的步骤数量。
- 中途离开前可点“暂存草稿”，草稿只保存在当前浏览器；下次新增菜谱时会询问是否继续。
- 熟悉所有字段后，可以点“切换到完整编辑”一次展开来源、版本、成本、工具等内容，来回切换不会清空已经填写的信息。
- 点击菜谱详情里的“编辑当前做法”可以修改当前版本；完整编辑模式下可以删除当前版本。
- 保存后，数据会先写入当前浏览器本地存储；如果页面顶部显示阿里云共享库已连接，也会同步到共享菜谱库。

## 同一道菜的不同做法

同一道菜想记录多个厨师或多个版本时，新增菜谱时把“菜名”写成一样即可。

例如都写“番茄炒蛋”，再分别填写：

- 厨师/来源：自用做法；做法版本/特点：家常快手版
- 厨师/来源：王刚参考版；做法版本/特点：饭店下饭版
- 厨师/来源：妈妈做法；做法版本/特点：少糖版

左侧列表会自动把同名菜合并成一组，右侧详情顶部会显示不同做法版本卡片。点击版本卡片，可以在不同厨师做法之间切换。

## 世界大类导航

菜谱列表上方有一排「世界大类」主导航，默认包含：现实中的饭、二次元中的饭、黑暗料理、存在于幻想中的饭。它和下方的场景标签（早餐/午餐/晚餐等）叠加过滤：先按世界大类筛，再按场景标签筛。

- 点击某个大类，只显示归属该大类的菜；点「全部」显示所有菜。
- 新增或编辑菜谱时，可在表单里选择「所属世界」，默认为“现实中的饭”。老数据没有该字段时也按“现实中的饭”处理。
- 点最右侧的「＋管理」按钮可以增删改大类：直接输入名称回车即新增；输入「删除 大类名」删除（默认 4 个大类不允许删除，可改名）；输入「改名 旧名>新名」重命名。
- 大类列表保存在当前浏览器本地。改名会同步更新已有菜谱的所属世界，但不会主动写回阿里云共享库，需要后续正常保存某道菜时才会把该菜的新归类同步上去。

## 按工具筛选菜谱

点击页面顶部的「我有什么工具」，会展开工具筛选面板：

- 面板里列出常见厨房设备（炒锅、汤锅、蒸锅、平底锅、油炸锅、烤箱、微波炉、空气炸锅、电饭煲、搅拌机、打蛋器、摇酒壶），勾选你厨房里有的工具，选中会高亮。
- 点「看看能做什么」后，按菜品所需工具分成两类：所需工具都齐（或本身无需特殊工具）的标「可以做」；只差 1 到 2 件工具的标「差一点」，并红字提示「还缺：XX」。缺 3 件及以上的不显示，避免干扰。
- 「可以做」的菜整卡可点，直接进入详情；「差一点」的菜卡上「进详情」和「问 AI 怎么替代」分开，点「问 AI 怎么替代」会打开右下角 AI 助手并自动询问缺失工具的替代方案（需先在 AI 助手里配置 API Key）。
- 勾选状态保存在当前浏览器本地，点「清空勾选」可全部取消。
- 菜谱详情页在基础信息下方会显示「所需工具」标签（该菜没有填工具时不显示）。新增或编辑菜谱时，可在表单「所需工具」里用空格分隔填写。

## AI 视频成谱工作台

点击页面顶部“AI 生成菜谱”，或在新增菜谱上方展开“已有视频或文字？交给 AI 智能填写”，即可进入统一的 AI 视频成谱工作台。

推荐流程：

1. 粘贴 B 站公开视频链接（支持常见 `bilibili.com/video`、BV/AV 地址和 `b23.tv` 短链接）。
2. 点击“获取视频信息与字幕”。页面会尝试获取视频标题、UP 主、简介和公开视频字幕。
3. 检查并修改字幕区内容。AI 不会直接观看视频，只会根据成功获取或手动粘贴的文字整理菜谱。
4. 展开“模型设置”，填写 API Key、OpenAI 兼容接口地址和服务商实际提供的模型 ID。DeepSeek 等兼容接口均可使用，但模型名称应以服务商控制台为准。
5. 点击“让 AI 整理成菜谱”，查看菜名、食材数、步骤数和摘要，再点击“一键填入菜谱表单”。
6. 保存前务必核对食材用量、火候、时间和成本，避免 AI 对口语字幕理解错误。

自动字幕只适用于视频本身提供公开字幕、且浏览器能访问 B 站公开接口的情况。如果视频没有公开字幕，或接口/跨域代理暂时不可用，工作台会保留链接和已获取信息，并提示手动粘贴字幕、简介、评论区教程或自己的听写内容。第一版不会下载视频、绕过登录权限，也不执行音频转写。

API Key 仅保存在当前浏览器的本地存储中，并由浏览器直接发送到你填写的兼容接口。公共电脑上不建议保存密钥。

## 可视化调整网页

点击页面顶部的“调整网站”，可以不用改代码，直接调整网页外观。

可以调整的内容包括：

- 网站标题
- 英文小标签
- 顶部说明文字
- 主色
- 背景色
- 卡片圆角
- 图片高度
- 左侧菜谱列表宽度

调整时页面会马上变化，刷新后也会保留。点击“恢复默认外观”可以把外观改回初始状态，菜谱内容不会被删除。

## 备份与迁移

- 点击“导出备份”会下载当前全部菜谱的 JSON 文件。
- 点击“导入备份”可以选择之前导出的 JSON 文件恢复数据。
- 页面顶部会显示同步状态；如果提示“阿里云 API 未配置”或“阿里云同步失败”，本次内容只保存在当前浏览器。
- 阿里云同步只覆盖菜谱数据，“调整网站”的外观设置仍保存在当前浏览器本地；换电脑前建议记录当前设置，或后续再升级为完整配置导出。
- 重要内容请定期导出备份，避免误删或覆盖后无法恢复。

## 后续上线说明

当前前端仍是纯静态网站，后续购买域名后，可以直接部署到静态托管平台；多人共享依赖阿里云函数计算 API 和 OSS。上线前建议把菜谱数据先导出备份，再确认函数权限、跨域配置和回滚方案。
