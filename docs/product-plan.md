# 我的食谱产品计划书

## 1. 我们想做什么

“我的食谱”不只是一个记录菜谱的小网页。它最终想做成一个大家一起维护的智能菜谱库：朋友、家人、同事看到好吃的做法，都可以顺手补进去；不知道今天吃什么时，可以直接从里面挑；看到一个做菜视频，也可以让 AI 帮忙整理成真正能照着做的菜谱。

现在它还处在内测阶段，但方向很清楚：先把网页做扎实，再做成手机上更好用的版本，后续再考虑微信小程序、iOS 和 Android。

一句话目标：

> 复制一个做菜视频链接，AI 帮你整理成菜谱；大家一起补充，慢慢变成我们自己的美食库。

## 2. 目前已经有的能力

当前网站已经具备这些基础功能：

- 菜谱新增、编辑、删除
- 记录菜名、来源、做法版本、介绍、食材、步骤、成本、耗时和适合时间
- 多人共同编辑，数据同步到 Firebase Firestore
- 右下角 AI 悬浮助手
- AI 辅助整理菜谱并一键填入表单
- B站链接识别和字幕抓取尝试
- “今天吃什么”随机推荐
- 购物清单汇总
- JSON 导入和导出备份
- GitHub Pages 在线访问

这些功能已经能支撑朋友之间小范围内测，但距离“好用、好看、稳定、愿意分享”还有距离。

## 3. 当前最需要解决的问题

### 3.1 稳定性

上线后的每一次修改都必须保证页面能正常交互，不能再出现脚本错误导致按钮失效的情况。

必须坚持：

- 每次修改 JavaScript 后执行语法检查
- 每次部署后检查控制台是否有红色报错
- 每次上线后至少验证新增、编辑、保存、AI 面板这几个关键动作

### 3.2 AI 体验

现在 AI 更像一个聊天框，用户还需要自己理解该怎么问。后续应该把它改成明确的任务入口：

- 粘贴视频链接生成菜谱
- 粘贴字幕或文案生成菜谱
- 根据现有食材推荐菜
- 优化已有菜谱

用户不应该研究 AI 怎么用，而应该点一个按钮就知道下一步该干什么。

### 3.3 美食氛围

现在页面更像一个管理工具，不够像美食网站。后续 UI 应该更有食欲感，更适合分享给别人。

需要突出：

- 菜品图片
- 今日推荐
- 食材标签
- 耗时、成本、适合场景
- 温暖、家常、有烟火气的视觉风格

## 4. A 计划：AI 视频生成菜谱

### 4.1 目标

把 AI 从“辅助聊天”升级成“视频菜谱整理助手”。

理想流程：

1. 用户点击“AI 生成菜谱”
2. 粘贴 B站、抖音、小红书或 YouTube 的视频链接
3. 系统尝试获取标题、作者、简介和字幕
4. AI 根据已有信息整理菜谱草稿
5. 用户确认或补充
6. 一键保存到菜谱库

### 4.2 第一阶段优先支持 B站

第一阶段优先做好 B站，因为目前已经有了基础识别逻辑。

需要完善：

- BV 链接识别
- b23.tv 短链接识别
- 标题、UP主、简介获取
- 字幕列表获取
- 字幕 JSON 解析
- 字幕失败后的人工粘贴引导

需要明确告诉用户：如果视频没有公开字幕，AI 不能凭空看视频画面，只能根据标题、简介、字幕或用户补充的信息整理。

### 4.3 AI 输出结构

AI 最终必须整理出统一格式：

```json
{
  "name": "菜名",
  "source": "来源 / 作者 / UP主",
  "variant": "做法版本",
  "description": "菜品介绍",
  "ingredients": [
    {
      "name": "食材名",
      "amount": "用量",
      "category": "主料 / 辅料 / 调味料",
      "note": "小白提示"
    }
  ],
  "cost": "预计成本",
  "occasion": "适合时间",
  "time": "预计耗时",
  "video": "视频链接",
  "steps": ["步骤1", "步骤2"]
}
```

### 4.4 AI 面板改造

AI 面板打开后不要只显示聊天输入框，而要先给用户几个明确入口：

- 粘贴视频链接生成菜谱
- 粘贴字幕或文案生成菜谱
- 根据食材推荐一道菜
- 帮我优化当前菜谱

生成结果也不要只输出一段 JSON，而要显示成菜谱草稿卡片：

```text
AI 已整理出菜谱：

菜名：蒜香鸡翅
耗时：约30分钟
成本：约25元
主料：鸡翅、蒜
调味料：生抽、蚝油、蜂蜜

[一键填入表单] [继续补充] [重新生成]
```

### 4.5 后续增强方向

后续可以继续支持：

- 小红书文案粘贴
- 抖音视频标题和文案粘贴
- YouTube 字幕
- 上传 .srt 字幕文件
- 图片识别菜谱
- 语音输入菜谱
- 自动生成购物清单
- 自动估算成本和耗时

## 5. B 计划：美食网站 UI 升级

### 5.1 目标

让别人一打开就感觉这是一个温暖、好吃、适合收藏的美食网站，而不是普通后台工具。

关键词：

- 温暖
- 家常
- 有食欲
- 手机友好
- 卡片式
- 图片突出
- 易分享

### 5.2 首页改造方向

首页应该更像一个美食入口：

```text
顶部 Hero 区
- 今天想吃点什么？
- 收藏你的家常菜、朋友菜谱和视频灵感
- 主按钮：AI 生成菜谱
- 次按钮：新增菜谱

今日推荐区
- 随机推荐一道菜
- 显示图片、耗时、成本、适合时间

菜谱分类区
- 早餐
- 午餐
- 晚餐
- 宵夜
- 快手菜
- 健身餐
- 下饭菜

菜谱卡片区
- 图片
- 菜名
- 来源
- 耗时
- 成本
- 标签
```

### 5.3 视觉风格

建议采用更有食欲的视觉语言：

- 奶油白背景
- 橙红色主按钮
- 番茄红强调色
- 深棕色标题文字
- 浅米色卡片
- 绿色健康标签
- 大圆角图片
- 柔和阴影

参考方向可以靠近：

- 下厨房
- 豆果美食
- 小红书美食收藏页
- 家庭食谱手账

### 5.4 菜谱卡片升级

新版菜谱卡片应该突出菜本身，而不是只显示文本。

卡片内容建议：

```text
[菜品图片]

番茄炒蛋
家常快手版

15分钟
约12元
晚餐 / 下饭菜

[查看做法]
```

如果没有图片，可以使用美食风格占位图或渐变背景，不要空白。

### 5.5 详情页升级

详情页应该像正式菜谱网站：

```text
大图
菜名
一句话介绍

标签：
15分钟 / 约20元 / 晚餐 / 家常菜

食材清单
- 主料
- 辅料
- 调味料

做法步骤
1. ...
2. ...
3. ...

视频链接
来源 / 厨师
```

### 5.6 手机端优先

未来要做微信小程序和 App，所以现在网页就应该按手机体验设计：

- 按钮足够大
- 卡片纵向排列
- AI 按钮固定在右下角
- 表单输入舒服
- 做法步骤清楚
- 图片优先展示

## 6. 多人协作规划

### 6.1 当前内测模式

当前采用所有人都能编辑的模式，适合内测。

好处：

- 不需要注册
- 朋友点开就能补菜谱
- 收集内容最快

风险：

- 任何人都能删改
- 暂时不知道是谁改的
- 误删恢复不方便

### 6.2 下一步增强

内测阶段可以先加这些轻量能力：

- 最近修改时间
- 修改人昵称
- 删除二次确认
- 操作日志
- 一键导出备份
- 一键恢复备份

正式阶段再做登录和权限。

### 6.3 正式权限模型

未来可以分三类角色：

| 角色 | 权限 |
| --- | --- |
| 管理员 | 添加、编辑、删除、恢复 |
| 成员 | 添加、编辑 |
| 访客 | 只读浏览 |

## 7. 微信小程序规划

### 7.1 为什么适合做小程序

菜谱非常适合小程序：

- 家人朋友容易打开
- 微信群里容易分享
- 做饭时手机查看方便
- 不需要下载 App
- 微信登录适合多人协作

### 7.2 小程序第一版功能

第一版可以先做：

- 首页菜谱列表
- 菜谱详情
- 新增菜谱
- 编辑菜谱
- AI 生成菜谱
- 今天吃什么
- 购物清单
- 分享给朋友

### 7.3 小程序注意事项

小程序正式版不能把 AI API Key 放前端，必须有后端代理。

视频解析也最好放后端做，因为小程序跨域限制更严格。

正式上架前还需要准备：

- 隐私政策
- 用户协议
- 数据删除说明
- AI 内容说明

## 8. iOS 和 Android 规划

不建议现在直接做原生 App。

更稳的路线是：

```text
Web 高质量版
    ↓
PWA 手机网页版
    ↓
微信小程序
    ↓
React Native / Flutter App
    ↓
iOS / Android 上架
```

等网站和小程序验证成功后，App 可以加入：

- 离线菜谱
- 拍照识别食材
- 语音输入菜谱
- 家庭成员共享
- 做饭计时器
- 每周菜单计划
- 营养分析

## 9. 技术架构规划

### 9.1 当前架构

```text
GitHub Pages
    ↓
HTML / CSS / JavaScript
    ↓
Firebase Firestore
    ↓
AI API
```

适合内测，成本低，上线快。

### 9.2 中期架构

未来需要后端：

```text
Web / 小程序 / App
    ↓
后端 API
    ↓
数据库
    ↓
AI 服务
    ↓
视频字幕解析服务
```

后端可以负责：

- AI Key 保护
- 视频字幕抓取
- 登录鉴权
- 权限管理
- 数据备份
- 操作日志
- App 和小程序统一接口

## 10. 阶段排期

### 第 1 阶段：稳定当前网站

目标：页面稳定可用。

任务：

- 修复所有 JS 报错
- 确认按钮交互正常
- 确认 Firebase 同步正常
- 清理重复字段
- 完善 README
- 建立基础验证习惯

预计：1 天

### 第 2 阶段：AI 视频生成菜谱

目标：粘贴视频链接后更顺畅地生成菜谱。

任务：

- 改造 AI 面板首页
- 增加“粘贴视频生成菜谱”入口
- 完善 B站字幕抓取
- 抓取失败时提供字幕粘贴区
- 输出菜谱草稿卡片
- 一键填入表单

预计：2 到 4 天

### 第 3 阶段：美食 UI 改版

目标：让网站更像美食产品。

任务：

- 首页 Hero 改造
- 菜谱卡片重设计
- 详情页重设计
- 分类标签
- 今日推荐模块
- 手机端适配优化

预计：3 到 5 天

### 第 4 阶段：多人协作增强

目标：让内测更安全。

任务：

- 修改时间
- 修改人昵称
- 删除保护
- 备份恢复
- 操作日志

预计：2 到 4 天

### 第 5 阶段：微信小程序预研

目标：为小程序做准备。

任务：

- 拆分数据接口
- 设计小程序页面结构
- 确定登录方式
- 确定 AI 后端代理方案
- 准备隐私政策

预计：1 到 2 周

### 第 6 阶段：App 化

目标：决定是否做 iOS / Android。

任务：

- 评估用户量和留存
- 确认核心功能
- 选择 Flutter 或 React Native
- 准备应用市场材料
- 完善账号体系和隐私合规

预计：后期执行

## 11. 对参与内测的人怎么说

可以这样邀请朋友：

> 我们在做一个朋友之间共用的菜谱库。你看到好吃的做法，或者家里常做的菜，都可以顺手填进去。以后不知道吃什么时，就直接从这里挑；看到视频菜谱，也可以让 AI 帮忙整理。现在还在内测，欢迎一起来补菜谱、提建议，把它慢慢做成真正好用的家庭美食库。

## 12. 最终愿景

最终它应该变成一个大家愿意长期使用的智能菜谱工具：

- 看到好菜，随手收进来
- 不知道吃什么，系统帮你挑
- 买菜前，一键生成清单
- 做饭时，手机打开步骤
- 家人朋友一起维护
- AI 帮忙整理和优化

它不是一个冷冰冰的管理系统，而是一个越用越有味道、越填越像家的美食库。
